前几天 DeepSeek 正式发布了 V4 预览版，这个新模型引起了不少关注。我也第一时间去试用了一下，分享一下我对这个模型的了解和一些感受。

<!-- more -->

## 百万上下文：这次真来了

DeepSeek-V4 最直观的升级就是上下文长度从之前的 128K 直接拉到了 **1M（一百万）**。这个量级什么概念？三体三部曲加起来也就不到一百万字，也就是说你可以直接把整本三体丢进去让模型分析。

这背后是 DeepSeek 新开发的注意力机制——**DSA（DeepSeek Sparse Attention）**。他们在 token 维度上做了压缩，结合稀疏注意力，大幅降低了对计算和显存的需求，让超长上下文不再是奢侈品。按 DeepSeek 的说法，1M 上下文以后会是 DeepSeek 所有官方服务的标配。

## 两个版本：Pro 和 Flash

这次 V4 沿用了类似 V3 的定位策略，提供了两个版本：

### DeepSeek-V4-Pro

旗舰版，各项能力拉满：

- **Agent 能力大幅提升**：在 Agentic Coding 评测中已经达到开源模型最佳水平。DeepSeek 内部员工已经在用 V4-Pro 作为日常的 Agentic Coding 模型，据说使用体验优于 Sonnet 4.5，交付质量接近 Opus 4.6 的非思考模式。
- **推理性能顶级**：在数学、STEM、竞赛型代码的测评中，超过了目前所有已公开评测的开源模型，跟世界顶级闭源模型掰手腕。
- **世界知识丰富**：大幅领先其他开源模型，仅稍逊于 Gemini-Pro-3.1。

### DeepSeek-V4-Flash

经济实惠的选择。世界知识方面比 Pro 稍弱，但推理能力接近 Pro。因为参数量和激活量更小，API 调用更便宜也更快速。在简单 Agent 任务上与 Pro 旗鼓相当，复杂任务上则有差距。

价格方面，Flash 的输入只要 1 元/百万 tokens，输出 2 元/百万 tokens，缓存命中更是低到 0.02 元。Pro 虽然标价高一些，但目前有 2.5 折优惠（到 5 月底），折后价格其实也很能打。

## Agent 能力专项优化

这可能是 V4 最让我感兴趣的部分。DeepSeek 专门针对主流的 Agent 产品做了适配和优化，包括 **Claude Code、OpenClaw、OpenCode、CodeBuddy** 等。在代码任务、文档生成等方面都有明显提升。

作为一个每天都在用 AI 辅助编程的人，模型在 Agent 场景下的表现其实比单纯的对话能力更重要。V4 在工具调用（Tool Calls）和多轮交互上的优化，让它在实际编码辅助场景中确实比前代好用很多。

## API 调用变化

如果你已经在用 DeepSeek API，需要注意几个变化：

1. **新模型名**：`deepseek-v4-pro` 和 `deepseek-v4-flash`
2. **旧名即将弃用**：`deepseek-chat` 和 `deepseek-reasoner` 将在 2026-07-24 停止使用，目前它们分别指向 V4-Flash 的非思考模式和思考模式
3. **思考模式**：支持 `reasoning_effort` 参数设置思考强度（high/max），复杂 Agent 场景建议用 max
4. **双接口兼容**：同时支持 OpenAI ChatCompletions 和 Anthropic 接口格式

## 开源与技术报告

DeepSeek 这次一如既往地选择了开源，模型权重已经上传到 Hugging Face 和 ModelScope，同时还发布了详细的技术报告。这一点确实让 DeepSeek 在开源社区的口碑一直不错——说到做到的开源承诺。

## 一些个人感受

上手试了几天，V4 的推理能力确实让人印象深刻。在复杂逻辑推理和代码生成上，能明显感觉到比 V3 系列又上了一个台阶。1M 上下文在实际使用中确实很爽——之前需要分段处理的超长文档，现在可以直接喂进去。

不过也需要注意，这毕竟还是预览版，有些地方还有打磨空间。比如在极复杂的 Agent 任务上，Pro 和 Flash 的差距还是比较明显的，如果对质量要求高，建议直接上 Pro 并开启思考模式。

总的来说，DeepSeek-V4 在推理、Agent 能力和上下文长度三个关键维度上都做到了行业前列，而且是开源的。考虑到它的 API 价格相比同级别的闭源模型要便宜得多，对于个人开发者和中小企业来说确实是非常有吸引力的选择。

期待正式版的表现。
